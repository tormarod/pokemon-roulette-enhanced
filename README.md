# pokemon-roulette-enhanced

A luck-based Pokémon browser game built with Angular. Spin roulettes to pick your starter, battle gym leaders and rivals, catch and evolve Pokémon, and try to beat the Elite Four and Champion. No backend — it's a static SPA that pulls Pokémon sprite/type/power data locally and renders everything client-side.

- A new Statistics page tracks activity across runs — victories, streaks, most-owned and signature Pokémon, your "nemesis" opponent, and more — persisted separately from the current run so it survives resets, with an explicit reset control of its own. It also tracks luck (are you landing Yes more or less than the odds implied?), a browsable run-history log with recent form and a win-rate trend chart, achievements with unlock toasts, a per-generation breakdown (runs/wins/streaks/records plus most-owned Pokémon, favorite types, and nemesis, filterable by generation), and lets you export/import your whole profile (stats, achievements, and Pokédex together) as a single JSON file or share a stats card image, with per-section resets alongside the full reset.
- A new opt-in **New Experience Mode** setting adds a pre-battle prep step to every battle (gym, rival, Elite Four, Champion): pick which team member leads the fight and optionally spend an X Attack or a potion before you see the odds, plus a choose-between adventure with a Danger meter and new threats between fights — see [New Experience Mode](#new-experience-mode) below. Off by default; existing players see no change in behavior unless they opt in.
- New Experience Mode also lets you **assign abilities to your Pokémon**: loot ability capsules from the adventure wheel and apply them from the PC to shift a battle's odds (a roster of 30, no longer locked to species), plus real **stakes on a rival loss**: your lead faints and needs a Revive to fight again, instead of the loss being free — see [Abilities & the faint mechanic](#abilities--the-faint-mechanic) below.
- Battles now show your exact win-chance percentage, plus a full breakdown of every Yes/No contribution (type matchup, round threat, bad omens, ability effects, X Attack) — on both the pre-battle prep screen and the spin screen itself, computed from the same numbers that build the wheel so the two can never disagree. See [Battle balancing](#battle-balancing) below.
- New Experience Mode gained a **coin economy and a Market** so the between-round choices aren't all dominated by catching: earn Coins from wins, some adventure cards, and a small per-round stipend, and spend them any time outside battle on potions, X Attacks, Ability Capsules, Revives, and Rare Candies. See [Economy & the Market](#economy--the-market) below.
- The Market now also sells the **Bicycle** (a pricier guaranteed extra threat-free adventure step, previously only found on the adventure wheel), and potion/Ability Capsule prices were rebalanced: Potion/Super Potion/Hyper Potion all cost more but stay correctly tiered (each larger potion is cheaper than the equivalent stack of smaller ones), while the random Ability Capsule got cheaper to offset its gamble.
- **Honey reworked into a target-share bias** (up to 3 types, ~55% of the wheel, stacking toward a 75% ceiling — see [Opponent agency](#opponent-agency) below) and is now **sold in the Market for 45 coins**, alongside its existing find-via-wheel path.
- **Repel and Max Repel reworked into New Experience-only threat-avoidance items**: instead of steering your next catch away from a type, Repel now keeps your very next adventure step threat-free and Max Repel covers the next 3 — see the Danger meter paragraph in [New Experience Mode](#new-experience-mode) below.
- **Economy & Market reconciliation**: Find Item and the Market are now disjoint — Find Item never hands you something you could just buy, and the **Bicycle moved back out of the Market** to a rarer lucky find. You can now **sell held items back for coins** from a new Market Sell section. The Market also gained a **finite per-run stock** — buying depletes each row, round advance doesn't refill it, and the only relief valve is a costly, escalating, capped paid **Restock**. The Toll Booth threat's shortfall spike was softened (still bites when you're short, just with a lower ceiling) so it no longer punishes spending as the game itself encourages — see [Economy & the Market](#economy--the-market) below.
- Tap any Pokémon in your team or PC storage to open its **evolution line**: every stage (including branches like Eevee's) in one view, with type/power detail per stage and any unlocked **Mega Evolution** forms appended at the end, locked with a padlock until you've actually Mega Evolved that species in battle.
- A **restyled roulette wheel**: every wheel now sits in a theme-aware card (dark/light) with a slim gold ring, flat pokeball hub, outlined labels, a harmonized slice palette, and an animated VICTORY/DEFEAT (or outcome) result chip; battle wheels show the opponent matchup and win-chance breakdown right on the card.
- The **team and item panels were restyled** to match: the sidebar is now a slim utility rail (trainer tile, badges, PC/Pokédex/Market) above a roster grid that shows each Pokémon's type and power right on the tile, plus a hover popover for its assigned ability; badges collapse into a click-to-expand popover instead of an always-open row. Items are grouped into Battle Aids, Field Items, Mega Stones, and Ability Capsules, with a name/description hover popover in place of native tooltips — Ability Capsules now show a distinct color per capsule (matching the ability's type) instead of one shared icon.

## Battle balancing

Every battle (gym, rival, Elite Four, Champion) resolves as a weighted Yes/No wheel spin — team power fills the Yes pool, and each Pokémon's own matchup against the opponent's type(s) shifts the odds. Advantage always adds to the **Yes** pool and disadvantage always adds to the **No** pool, so a bad matchup shows up as visibly more red on the wheel, not a smaller green wedge — every Pokémon keeps its full power in green regardless of matchup. The No pool also grows with how far you've progressed (`ceil(round × 1.25)`, added as a flat "round threat"), so the run gets harder toward the Elite Four and Champion — tuned so the late game stays a real fight without being a coin flip for a well-built team.

There's no move/ability system — a Pokémon only has its one or two static types — so the model scores offense and defense **symmetrically**: every one of a member's types is always "active" on both sides, with no cherry-picking a single best attacking type. For each of the member's types against each opponent type, a super-effective hit is `+1`, an opponent that resists/is-immune-to that attack is `-1`/`-2`; symmetrically on defense, resisting/being immune to the opponent's hit is `+1`/`+2`, being hit super-effectively is `-1`. Summed across every type pair (with a repeated opponent type — e.g. Elite Four member Lance's `['dragon', 'dragon']` — counted twice, an intentional emphasis lever, not a data typo), this gives a single signed **net score** per member: positive is a net advantage, negative a net disadvantage, zero neutral. A member's own type against the identical opponent type always cancels to exactly zero (its offensive and defensive halves are mirror images with opposite sign) and is excluded from the on-screen breakdown, so a type never gets credited with "resisting" an opponent it's also being resisted by right back.

The net score is converted to a ticket delta by multiplying by a per-Pokémon unit, `ceil(power / 2)` — never zero, so even a power-1 Pokémon's matchup always matters, and it depends only on that one Pokémon's own power, never on team size or which other Pokémon are on the roster. Because both offense and defense count, a type pair that's favorable both ways (e.g. Water hits Fire super-effectively *and* resists Fire's counter-hit) scores higher than one that's only favorable on one side — and conversely, an unfavorable pair that cuts both ways (e.g. Fire beats Grass and resists Grass's counter) is a harsher penalty than a plain one-sided weakness. A purely defensive shortcut like immunity no longer unconditionally wins, either — it can be offset by a genuinely bad matchup against a different one of the opponent's types.

Both of these numbers (the `1.25` round-threat multiplier and the `ceil(power / 2)` matchup unit) were doubled/eased together in a single balance pass: base power caps at 5, so a full team's type-countering headroom was too small to meaningfully fight back against a No pool that grew unbounded with round — the late game was drifting toward a coin flip regardless of team quality. Lowering the round multiplier (from `1.5`) gives everyone flat relief, while doubling the matchup unit (from `ceil(power / 4)`) is pure agency — a well-countered team pulls meaningfully ahead, but the same symmetric scoring means a genuinely mismatched team now pays a steeper price too.

The full calculation lives in [`TypeMatchupService`](src/app/services/type-matchup-service/type-matchup.service.ts), shared by all four battle types via [`BaseBattleRouletteComponent.buildVictoryOdds()`](src/app/main-game/roulette-container/roulettes/base-battle-roulette/base-battle-roulette.component.ts).

## New Experience Mode

New Experience Mode is an opt-in setting (Settings screen) that adds a pre-battle prep step and higher stakes to every battle — gym, rival, Elite Four, and Champion alike. It's off by default (Classic mode); toggling it only takes effect on your *next* run, and whichever value was in effect when a run started stays locked in for that run's whole duration, so you can't dodge a bad matchup — or get surprised by a mechanic you didn't opt into — by flipping the setting mid-run.

With it on, every battle opens with a prep panel instead of going straight to the odds wheel:
- **Choose your lead**: pick one team member to lead the fight. That Pokémon's own type-matchup delta (see [Battle balancing](#battle-balancing) above) is applied a *second* time on top of the whole team's total — a favorable lead swings the Yes pool further, an unfavorable one adds more to the No pool. Every team member's live delta is shown before you pick, so it's an informed read of the matchup, not a blind choice — and picking is mandatory every battle, keeping that read-the-opponent moment in front of you each time.
- **Pre-spin X Attack**: optionally spend an X Attack *before* the wheel spins, rather than only reactively after a loss. X Attack becomes a one-time, consumed power boost (instead of Classic mode's quirk of every X Attack in your bag passively applying every battle, forever, without being used up); the boost is your team's mean power plus the current round, so it keeps pace with the round-threat No tickets and stays worth using late-game. Potions aren't part of the prep — they're already used automatically (weakest first) as a retry the moment a spin lands No, so there's nothing to gain by committing one up front.

Both the lead pick and any item use are a single editable draft — change your mind freely — until you hit Confirm, which commits everything atomically. From that instant it's persisted the same way a wheel spin's outcome is: a reload mid-prep re-shows the same draft state, and a reload after Confirm skips straight back to the already-computed odds, never a fresh roll.

Rival battles gained the same prep step (lead pick, pre-spin X Attack, once-per-battle ability free-retry) as part of New Experience Mode specifically — but, unlike gym/Elite Four/Champion, a rival loss **cannot** be salvaged with a potion. Potions are reserved for gym, Elite Four, and Champion battles; a rival loss is final and faints your lead (see below), keeping rivals a genuine stakes moment rather than something you spend your way out of. Classic-mode rival battles are unaffected and stay win/loss-only, exactly as before.

With New Experience Mode on, the adventure step between fights also changes from a passive wheel spin, paced by a **Danger meter** that stays visible throughout the whole roster-building stretch between fights — not just on the step where it's rolled — so its trend is always in view; it's hidden during actual battles (which have their own odds) and before the adventure starts. Hover or tap it for a tooltip explaining the mechanic. The meter climbs with rounds survived, drops sharply whenever a threat is drawn, and recovers a little faster on safe steps than it used to; three threats in a row force the next step safe (a hard pity), so danger never spirals unrecoverably. Any reward that hands you extra adventure steps — the **multitask** reward (two extra steps) or the **Bicycle** item (one extra step) — makes those bonus steps guaranteed threat-free (the meter still climbs across them), so a reward can never be the thing that ambushes you. **Repel** and **Max Repel** (New Experience-only items, usable proactively from the item bar at any time) work the same way from the player's side: Repel shields your very next adventure step from a threat, Max Repel shields the next 3. Like the reward-granted steps, a Repel/Max Repel shield is delay-only — danger keeps climbing underneath and is never cooled by walking past an encounter, so it can't be used to stall the meter down. On a **reward** step, 3 candidate outcomes are drawn and you pick one — the familiar catch, trade, legendary encounter, Team Rocket, etc. On a **threat** step, there's no picker: a single eligible outcome is drawn at random from a ten-entry pool and its effect just happens to you, with no way to steer toward the least-bad option — a Team Rocket ambush (the same steal/run/defeat mini-wheel, framed as a threat instead of a gamble), an item theft (loses one random item, or finds nothing with an empty bag), a bad omen (extra risk carried into your very next battle only, then it clears), a Forced Retreat (benches a weak-biased team Pokémon to PC storage, locked there — undraggable — until your next real battle resolves), a Spooked scare (spikes the Danger meter back up, no item/Pokémon cost), a Marked Target (one random team Pokémon is forced to lead your next battle's prep screen, every other pick disabled), a Poké Ball Malfunction (your very next catch attempt has a chance to fail and the Pokémon gets away), a Toll Booth (a round-scaled Coin toll — pay what you can afford; coming up short on top of that spikes the Danger meter, scaled to how much went unpaid), a Scouting Report (your next real battle's opponent gains one extra type, picked at random from the counters to your strongest Pokémon across team and PC, locked in at draw time so benching that Pokémon afterward is a real trade-off, not a free dodge), and a PC Lockout (freezes the PC both directions — no withdraw, no deposit — until your next real battle resolves). Threats that would be a no-op given your current roster (e.g. Forced Retreat or PC Lockout with too few Pokémon to spare) are filtered out of the draw rather than wasted. Classic mode's adventure step is unaffected — still a single wheel spin.

### Abilities & the faint mechanic

Also New Experience Mode-only: **abilities are something you build toward, not something you roll**. Abilities are no longer tied to a species — instead, a **roster of 30** is looted as **ability capsules**. "Find an Ability Capsule" is one of the adventure wheel's reward outcomes; landing it opens a capsule wheel that awards one specific capsule (e.g. "Blaze", "Intimidate", "Sturdy") into your bag. You then **assign it from the PC**: open a team or stored Pokémon and apply a capsule you're holding, which shows an ability badge on that Pokémon (hover or tap it for what the ability does) and, in battle, nudges the Yes/No odds. Assignment is re-assignable: applying a new capsule overwrites the old ability and consumes the capsule, so it's a real, ongoing choice about who gets what. An assigned ability carries through evolving that Pokémon, and through a mega evolution's active-form window. Effects range from flat odds shifts to conditional ones (bonuses that only fire on a favorable or unfavorable matchup, team-type synergy, dual-/single-type payoffs, capped "snowball" and "comeback" scalers, and a couple of non-odds effects — a free retry, and lead faint-immunity). Capsules never appear on the regular item wheel, and Classic mode never has an active ability, regardless of team composition.

Rival battles specifically also gained real stakes: a rival loss used to be free (the run just continued). Now, on a loss with no retries left (rivals can't be bailed out with a potion — see above), your chosen **lead Pokémon faints** — it's immediately pulled from your team (freeing a slot) and moved to the PC, shown greyed-out with a "Fainted" badge and undraggable back into the team until revived with a **Revive** item (a new, rarer-than-Hyper-Potion find). If fainting would leave your team empty, the run ends there, same as any other loss. A lead you've assigned the **Sturdy** ability survives a would-be faint instead, once per battle. Gym, Elite Four, and Champion losses are unaffected — a loss there still ends the run outright, exactly as before.

## Economy & the Market

Also New Experience Mode-only. The between-round choices used to be lopsided: **catching a Pokémon was almost always the strongest pick** until your team was six-strong, because team power is the one durable lever that fills the Yes pool — every other reward was a consumable or flavor. A coin economy fixes that by giving every choice a way to feed into player power.

- **Earning Coins.** A live counter sits in the team strip (it pulses with a "+N" floater whenever it climbs). Coins come from three sources: a round-scaled drop on every battle win (gym, rival, Elite Four), a small flat **per-round stipend** folded into the round-advancing wins (so it lands exactly once per round and can't be farmed by the multitask reward), and a bonus on several non-catch adventure cards (Trainer Battle, Explore Cave, Snorlax, Go Fishing, Find Fossil) — so picking one of those instead of a catch is no longer wasted value. The old fake "Buy Potions" card (which just handed you a free potion) is repurposed into a "Found Coins" bundle. Champion wins pay nothing — the run's over, so there's nothing left to spend on.
- **The Market.** An always-available shop (a button in the team strip, like the PC) that you can open any time **outside** combat — it never advances the round. It carries consumed/assigned staples only — potions (Potion/Super/Hyper), an X Attack, a Revive, a Rare Candy, a random **Ability Capsule** (bagged like a found capsule, then assigned from the PC), and **Honey** (the type-share coverage lever) — plus a **Sell** section for turning held items back into coins (see below). The **Bicycle** is not sold here; it's a permanent held multiplier, so it stays a rarer, luckier find on the Find Item wheel instead. Rows you can't afford, or that are sold out (see Market stock below), are disabled. It's deliberately closed during a battle spin or the prep panel, so a purchase (or sale) can never react to a shown loss or to odds you've already seen — the same anti-reroll fairness rule the rest of the run persistence follows.
- **Find Item, disjoint from the Market.** The Find Item wheel never hands you something you could just buy — it's filtered to exclude every item the Market stocks (derived automatically from the Market's stock list, so the two can't drift apart), leaving only find-only gadgets (Exp Share, Escape Rope, Repel, Poké Radar, Max Repel, Link Cable) and the **Bicycle**. Bicycle is a permanent, run-long power-item (an extra threat-free adventure step every round) rather than a consumable, so it's deliberately kept as a find rather than sold — buying one outright would undercut the stock system's scarcity.
- **Selling items.** Any held item you don't need converts back to coins from the Market's Sell section — Market-sold consumables and Ability Capsules sell for a fraction of their buy price, find-only gadgets (including Bicycle) sell for a flat rate, and mega stones aren't sellable (they're build pieces, not economy fodder). Selling is always worth strictly less than buying, so there's no buy/sell arbitrage loop — it's a release valve for dead-weight items (a bias item found with a full, settled team, say), not a coin faucet.
- **Market stock.** Every Market row has a **fixed per-run stock cap** — you start each run at full capacity, and buying depletes it. Stock does **not** refill automatically on round advance; the only way to replenish it is a paid **Restock** (a confirm-gated action in the Market) that tops every row back up to its cap. Restock is deliberately a rare, expensive relief valve rather than a spam button: its price escalates each time it's used within a run, and it's capped at a handful of uses total. This makes coins a genuine constraint (rather than something you can convert into unlimited retries) while still guaranteeing you're never permanently locked out of potions for the rest of a run.

All the coin amounts, Market prices, sell rates, stock caps, and restock pricing are tunable dials in [`economy-config.ts`](src/app/main-game/roulette-container/economy-config.ts). Classic mode has no coins, Market, or item selling — it's unchanged, and Find Item there is the full, unfiltered item pool exactly as before.

## Opponent agency

Team composition used to be fully locked in by catch/trade RNG before the upcoming opponent's type was ever revealed — the reveal only happened once the battle screen itself mounted, well after any roster-building decisions were made. A few additions give the player real, informed agency over that process without touching the core RNG loop or the [battle-odds math](#battle-balancing) itself:

- **Opponent preview**: a persistent banner shows the next gym leader's type (or, once you reach `elite-four-preparation`, the first Elite Four member's) throughout the roster-building stretch that precedes them, so swapping team members or using a bias item below is an informed choice instead of a guess. It's hidden during actual battles (which reveal their own opponent) and before the adventure starts. A handful of rounds resolve to one of several possible types only once the battle begins (e.g. gym leader trios); the preview shows every possible type for those rounds rather than falsely implying certainty.
- **Type-bias items**: Poké Radar (found via the wheel) hard-guarantees your next new Pokémon's type. It applies to any wheel that hands you a new Pokémon (catch, trade, fossil, legendary, cave, fishing, mysterious egg, or Area Zero), and the type is picked by clicking it directly — no RNG in the pick itself, shown in its own indicator next to the Items panel. You can use it while an obtain wheel is already on screen — a type-picker opens over the wheel without leaving it, and the wheel's odds redraw live once you pick — instead of only being usable from the Items panel between screens. It's single-wheel-use: it weights exactly the next (or current) obtain wheel's resolution, then clears immediately, so it can never carry over and benefit a second wheel in the same stretch. It also clears automatically on reaching a battle, as a safety net for a bias that was set but never actually used. The effect is also legible on the wheel itself before you spin: toward-type slices get a bright outline, on every wheel that can hand you a new Pokémon.
- **Honey** (common, granted at the start of every run) works on a different mechanic: a **target share** rather than a fixed multiplier. You pick **up to 3 types** (a toggle-to-select picker with a Confirm button once you've chosen at least one; picking just 1 behaves like a single instant pick), and together they're made to occupy about **55% of your next wild Pokémon wheel** — split across however many you picked — recomputed live from whatever's actually in the pool at spin time, so the effect holds steady whether the pool is small (early rounds) or large (late rounds), unlike a flat weight bonus which drifts with pool size. Uses stack toward a hard ceiling: a 2nd Honey pushes the combined share to ~70%, further uses approach but never exceed **75%** — that ceiling is deliberate, so no amount of Honey can replicate a Poké Radar's absolute guarantee. Honey shares the click-to-pick UX of Poké Radar (usable on a wheel already on screen, clears after one use, highlighted on the wheel).
- **Link Cable** (rarer still) triggers a trade encounter on demand instead of waiting for the adventure wheel to offer one.
- **Trade-out is a direct pick**: choosing which of your *own* Pokémon to offer in a trade is a clickable grid, not a wheel spin — nothing was ever being won by randomizing a choice among Pokémon you already own. Evolution choices and a Team Rocket encounter's steal target stay wheel-based, since those still involve genuine randomness.
- **Team Rocket's steal odds are weighted by power**: a stronger Pokémon puts up more of a fight, so it's harder for Team Rocket to steal than a weak one — previously every team member was equally likely to be taken.

## Run persistence

The game auto-saves your run to `localStorage` as you play — team, PC storage, items, badges, round, current screen, New Experience Mode's coin balance, and the transient mega-evolution battle state — via [`RunPersistenceService`](src/app/services/run-persistence-service/run-persistence.service.ts). On startup, if a save exists, it's restored before the first screen renders, so a reload or closed tab drops you back where you left off instead of wiping the run. The save is cleared automatically once a run ends (win or loss) or is restarted. (Persisting the mega battle state matters: a mega form is baked into the saved team, so without also saving what to revert it back to, reloading mid-mega would strand the Pokémon in its mega form permanently — and a stuck mega blocks any further mega evolution.)

Saving happens on every committed change (team/item/badge/state mutation), not on a timer or only at major checkpoints — this matters for fairness: a save that only happened between battles would let a player spend a potion for extra retries, reload before the battle resolves, and get the potion back for free. Auto-saving on every committed mutation closes that off, since a reload can never rewind past something that already happened.

The wheel spin itself gets the same treatment via a separate, lightweight [`PendingSpinService`](src/app/services/pending-spin-service/pending-spin.service.ts): the instant you click spin, the outcome is decided and committed — before the multi-second reveal animation plays, not after. Reloading mid-animation doesn't offer a fresh roll; the next load immediately resolves to the same outcome you already locked in. What's *not* saved (mid-battle retry count from an unconsumed potion, other in-progress on-screen choices) can only be forfeited by reloading, never regained — reloading mid-fight loses you the extra retries a potion bought, but the potion itself stays spent.

## Development server

To start a local development server, run:

```bash
npm start
```

This runs `ng serve --host 0.0.0.0 --port 4200` (see `package.json`), binding to all network interfaces so you can also test from another device (e.g. a phone) on the same network via your machine's local IP. Once the server is running, open your browser and navigate to `http://localhost:4200/`. The application will automatically reload whenever you modify any of the source files.

## Building

To build the project run:

```bash
npm run build
```

This compiles the project and stores the build artifacts in `dist/pokemon-roulette-enhanced/`. By default, the production build optimizes the application for performance and speed. Use `npm run watch` for an unoptimized development build that rebuilds on file changes.

## Running unit tests

To execute unit tests with the [Karma](https://karma-runner.github.io) test runner, use the following command:

```bash
npm test
```

## Deploying

Every push to `main` automatically builds and deploys to GitHub Pages via the `.github/workflows/deploy.yml` GitHub Actions workflow (gated behind the test suite passing). This requires a `GH_TOKEN` repository secret (a personal access token) to be configured, since GitHub Pages rebuilds aren't triggered by the default `GITHUB_TOKEN` on public repos.

To deploy manually instead, using [angular-cli-ghpages](https://github.com/angular-schule/angular-cli-ghpages):

```bash
npm run deploy
```

This runs `ng deploy --base-href=/pokemon-roulette-enhanced/`, which builds the app and pushes it to the `gh-pages` branch.
